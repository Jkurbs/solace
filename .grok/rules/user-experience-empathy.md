# User experience — deep empathy

When working on **any user-facing** Solace/Hermes surface (dashboard, onboarding, capital, sign-in, access messaging):

1. Read and follow **`notes/user-experience-empathy.md`**.
2. North star: *This product should make a careful person feel carefully treated.*
3. State the emotional job of the screen before layout or copy changes.
4. Prefer one clear next step over dense metrics in empty/waiting states.
5. Never present infrastructure failures (rate limits, SMTP, bridge lag) as user rejection.
6. Keep Simulation / illustrative / young-sample honesty — honesty is care.
